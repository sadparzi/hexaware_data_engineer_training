1. What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment (or Continuous Delivery). It is a DevOps practice that automates the process of building, testing, and deploying software.

Instead of manually building and deploying the application every time a developer makes a change, CI/CD pipelines perform these tasks automatically.

Continuous Integration (CI)

Continuous Integration is the practice of automatically integrating code changes into a shared repository.

Whenever a developer pushes code to Git:

The code is downloaded.
The application is built.
Unit tests are executed.
Code quality checks are performed.
A deployable package (artifact) is created.

If any step fails, the pipeline stops and the developers are notified.

Continuous Deployment (CD)

Continuous Deployment automatically deploys the successfully built application to an environment.

Faster software delivery
Automatic build and testing
Early detection of bugs
Reduced manual effort
Consistent deployment process
Faster feedback to developers
Improved code quality
Reduced deployment errors
Easy rollback if deployment fails
Faster release cycles
Better collaboration between development and operations teams
